## What is Pinfold?

Most KML files come from somewhere else — Google Earth exports, GIS tools, a route a guide shared, a colleague's site survey. They end up buried in email attachments and downloads folders. **Pinfold gives those files a permanent, organized home on your device**, and lets you browse and navigate them completely offline.

It's a faithful viewer and launcher: it reads your files exactly as they were authored, then hands you off to whichever navigation app you prefer.

## Features

- **Built for offline** — load a file once with a connection; icons and photos it references are cached on import, so they're there when you're off the grid. Nothing depends on a server.
- **Browse your places** — navigate the original folder hierarchy from the KML, tap any placemark for its full details (description, photos, extended-data fields), with custom icons and colors from the file's styles preserved.
- **See it on the map** — view all of a file's pins together on an embedded map; nearby pins group into numbered clusters when you zoom out so dense files stay readable.
- **Navigate with the app you already use** — open any place in Apple Maps, Google Maps, Waze, Organic Maps, Guru Maps, OsmAnd, Magic Earth, Citymapper, and more. Set a default for one-tap navigation, or pick each time. Apps you don't have installed are hidden automatically.
- **Import from anywhere** — share a `.kml`/`.kmz` from any app, open one directly from Files or an attachment, or import several at once.
- **Optional iCloud sync** — turn on iCloud to keep your imported files on every device signed in to the same account. Files stay in your private iCloud; Pinfold has no account and no servers.
- **Private by design** — no account, no sign-up, no ads, no tracking. Your location is shown only on your device and never leaves it.

## Building & running

The Xcode project is **generated by [XcodeGen](https://github.com/yonaskolb/XcodeGen)** from `project.yml` and is gitignored — never edit or commit `Pinfold.xcodeproj` by hand.

```bash
# 1. Clone
git clone <repo-url> && cd kmlore

# 2. Generate the Xcode project from project.yml
xcodegen generate

# 3. Open in Xcode and run, or use the scripts below
open Pinfold.xcodeproj
```

After adding, removing, or renaming files under `App/`, `ShareExtension/`, or `AppTests/`, run `xcodegen generate` again (the scripts do this for you).

### Scripts

| Script | What it does |
|---|---|
| `scripts/build.sh` | Builds the app for the iPhone simulator (code signing disabled) |
| `scripts/test.sh`  | Runs the app test suite (filter with `scripts/test.sh -only-testing:PinfoldTests/<Suite>`) |

> **Note:** The App Group / Share-Extension handoff and iCloud sync only work on a **signed** build (device, Mac, or TestFlight). They are inactive on the unsigned simulator builds the scripts use.

## Architecture

Pinfold is split into two parts:

- **`PinfoldCore/`** — a standalone Swift Package with no UIKit/SwiftUI/SwiftData dependency: the KML/KMZ parser. Entry point is `KMLReader.read(data:) -> ParsedKML`. Depends only on [ZIPFoundation](https://github.com/weichsel/ZIPFoundation) (for KMZ archives).
- **`App/` + `ShareExtension/`** — the SwiftUI app (Liquid Glass UI), using **MV + `@Observable`** (no view-model layer).

The catalogue is **folder-as-source-of-truth**: each imported file lives in its own folder (under Application Support, or in iCloud Drive when sync is on) alongside a small metadata sidecar. The parsed KML tree is never persisted — it's **re-parsed from disk on open**. Derivable per-device resources (extracted KMZ media, downloaded remote icons/photos) are cached locally and rebuilt on each device from the synced original.

## Project structure

```
.
├── App/                  # The SwiftUI app (Model, Views, Services, Support, Maps)
├── ShareExtension/       # Share-sheet extension that drops files into the shared inbox
├── AppTests/             # App test suite (Swift Testing)
├── PinfoldCore/          # Standalone KML/KMZ parser package (+ its own tests)
├── samples/              # Sample .kml / .kmz files for testing
├── scripts/              # build.sh, test.sh
├── docs/                 # Design specs, plans, icon assets
├── project.yml           # XcodeGen source of truth (generates Pinfold.xcodeproj)
└── .swiftlint.yml        # Lint config
```

## Testing

```bash
# App tests (iPhone simulator, code signing disabled)
scripts/test.sh

# Parser package tests
cd PinfoldCore && swift test
```

## Formatting & linting

- **Format:** `swiftformat` (config-less, pinned via `.swift-version`).
- **Lint:** `swiftlint lint` — kept at **0 errors**.

## Privacy

Pinfold collects **no data** and does **no tracking**. All your files live on your device and, if you enable sync, in your own private iCloud. See `App/PrivacyInfo.xcprivacy` for the declared API-usage reasons.
